# 架构文档

## 目录结构
```
sightly/
├── main.py                    # 入口
├── core/                      # 核心基础设施层
│   ├── state.py               # AppState—集中状态管理（信号驱动）
│   ├── workspace.py           # WorkspaceManager—文件 I/O
│   ├── config.py              # ConfigManager—配置读写
│   ├── logging.py             # LoggingManager—三级日志
│   ├── priority_lock.py       # PriorityLock—优先级互斥锁
│   ├── click_handler.py       # ClickWorker—统一鼠标点击
│   └── async_utils.py         # 异步工具函数
├── modules/                   # 后端功能模块（Worker+Manager）
│   ├── ocr.py / timed.py / number.py / image.py
│   ├── color.py / background.py / script.py / alarm.py / input.py
├── input/                     # 输入后端（策略模式）
│   ├── base.py / controller.py / dd_input.py / pyautogui_input.py / win32_input.py
├── utils/                     # 通用工具（Recognizer/Manager）
│   ├── screenshot.py / recognition.py / window_capture.py
│   ├── quick_switch.py / coordinate.py / image.py
├── ui/                        # UI 层（PySide6）
│   ├── main_window.py / theme.py
│   ├── home_panel.py / ocr_panel.py / timed_panel.py / ...
│   └── components/            # 可复用组件
├── config/ workspace/ logs/ voice/ icon/ drivers/
```

## 核心架构
```
MainWindow
  ├── AppState(信号中枢)  ←→  UI Panels
  ├── ConfigManager / WorkspaceManager / LoggingManager
  └── Modules(功能模块) → InputController / ScreenshotManager / Recognition Utils
      优先级锁: Number(6) > Timed(5) > Image(4) > OCR(3) > Color(2) > BG(1)
```

## 数据流
```
UI → Signal → AppState → Signal → MainWindow → Module/Worker.start()
Worker → ScreenshotManager(priority lock) → Recognition → ClickWorker/KeyEventWorker
```

## 配置持久化
```
保存: Panel.collect_config() → AppState.save_current() → WorkspaceManager.save()
加载: WorkspaceManager.load() → AppState._load_workspace() → config_loaded Signal → Panel.set_config()
```

## 核心类职责
**core/**
- AppState: 集中状态管理，Signal 广播状态变化
- WorkspaceManager: workspace/ 目录文件 I/O
- ConfigManager: JSON 配置读写
- LoggingManager: 三级日志（debug/log_message/error）
- PriorityLock: 堆实现优先级互斥锁
- ClickWorker: 鼠标点击执行器（随机偏移+坐标验证）

**modules/** — 每个模块按 Worker+Manager 双层结构：
- Worker: 执行单元（OCRWorker, TimedTask, NumberWorker, ImageDetectionWorker 等）
- Manager: 生命周期管理（ImageDetectionManager, ColorRecognitionManager, BackgroundManager）

## 设计模式
1. 集中状态管理+信号驱动通信 — AppState 通过 Qt Signal 松耦合
2. 工作空间系统 — 多空间隔离配置
3. 优先级锁 — 堆实现，确保高优先级模块优先获取资源
4. 输入后端策略模式 — InputManager 工厂选择 DD/PyAutoGUI/Win32
5. 识别引擎统一化 — utils/recognition.py 提供统一识别器类
6. Worker+Manager 双层 — 每个功能模块拆分为执行单元和生命周期管理
